# Как работать с GitHub

> Краткая инструкция для участников проекта **2026_kidbook** — нашей википедии для восьмиклассников.

---

## 1. Issues — предлагаем идеи и сообщаем о проблемах

**Issue** - это что-то вроде заявки или записки для команды. Через них мы обсуждаем новые статьи, предлагаем правки и сообщаем об ошибках.

### Как создать Issue

1. Откройте репозиторий: [github.com/lemito/2026_kidbook](https://github.com/lemito/2026_kidbook).
2. Перейдите во вкладку **Issues** (в верхнем меню, рядом с «Code»).
3. Нажмите зелёную кнопку **New issue**.
4. Заполните два поля:
   - **Title** — короткий и понятный заголовок. Например: `Новая статья: Солнечная система` или `Ошибка в статье про фотосинтез`.
   - **Comment** — подробное описание. Расскажите, что именно вы предлагаете или что нашли.
5. Нажмите **Submit new issue**.

### Как оформлять Issue правильно

Старайтесь придерживаться простого шаблона:

```
**Что предлагаю:**
Написать статью про Древний Египет.

**Почему это нужно:**
Тема входит в программу 8 класса по истории, а статьи у нас ещё нет.

**Примерный план:**
- Географическое положение
- Фараоны и пирамиды
- Повседневная жизнь египтян
```

### Полезные советы по Issues

- Если хотите взять задачу себе — напишите комментарий «Беру!» под нужным Issue, чтобы другие знали.
- Используйте **Labels** (метки), если они настроены в репозитории, — например, `новая статья`, `ошибка`, `улучшение`.
- Чтобы сослаться на Issue в другом месте, используйте `#номер` — например, `#12`.

## 2. Fork + Pull Request — вносим изменения в проект

Вы не можете редактировать основной репозиторий напрямую (и это правильно — так мы защищаем проект от случайных ошибок). Вместо этого используется схема **Fork → Правка → Pull Request**.

### Шаг 1. Делаем Fork (свою копию)

1. Откройте репозиторий: [github.com/lemito/2026_kidbook](https://github.com/lemito/2026_kidbook).
2. В правом верхнем углу нажмите кнопку **Fork**.
3. GitHub создаст копию репозитория в вашем аккаунте. Теперь у вас есть `ваш-логин/2026_kidbook` — это ваша личная песочница, в которой можно делать что угодно.

### Шаг 2. Вносим изменения

Проще всего работать прямо на сайте GitHub:

1. Откройте **свой** форк (`ваш-логин/2026_kidbook`).
2. Перейдите в нужную папку (например, `статьи/`).
3. Чтобы **создать новый файл** — нажмите **Add file → Create new file**.
4. Чтобы **отредактировать существующий** — откройте файл и нажмите на иконку карандаша (✏️).
5. Внесите изменения. Не забудьте про формат — пишите в Markdown (см. инструкцию «Как писать статьи»).
6. Внизу страницы заполните поле **Commit message** — кратко опишите, что вы сделали. Например: `Добавил статью про Солнечную систему`.
7. Нажмите **Commit changes**.

### Шаг 3. Создаём Pull Request (предлагаем изменения)

Теперь нужно «отправить» ваши изменения из форка в основной репозиторий:

1. Откройте **свой** форк на GitHub.
2. Вы увидите плашку наверху: *«This branch is X commits ahead of lemito:main»*. Нажмите **Contribute → Open pull request**.
3. GitHub покажет, какие именно изменения вы предлагаете. Проверьте, что всё выглядит правильно.
4. Заполните описание Pull Request:
   - **Title** — что вы сделали. Например: `Добавлена статья: Солнечная система`.
   - **Description** — подробности. Можно сослаться на Issue: `Closes #5` (это автоматически закроет Issue №5 после принятия).
5. Нажмите **Create pull request**.

После этого кто-то из команды проверит вашу работу (сделает **review**) и либо примет изменения (**merge**), либо попросит что-то доработать в комментариях.

### Шаг 4. Синхронизируем форк (если проект обновился)

Основной репозиторий обновляется, пока вы работаете. Чтобы ваш форк не отставал:

1. Откройте **свой** форк на GitHub.
2. Если он отстаёт, вы увидите: *«This branch is X commits behind lemito:main»*.
3. Нажмите **Sync fork → Update branch**.

Готово — теперь у вас актуальная версия проекта.

## Шпаргалка: весь процесс в двух словах

```
1. Нашли ошибку или есть идея?  →  Создайте Issue
2. Хотите сами написать/поправить?  →  Сделайте Fork
3. Внесли изменения в своём форке?  →  Откройте Pull Request
4. Дождитесь проверки  →  Готово! 
```
